Three-phase repo identification for each of the 50 CodeQA contexts:
1. URL extraction -- regex-scans context for github.com/owner/repo, picks most frequent, validates via PyGithub.
2. README heading -- looks for =# Name= headings where name appears in import/from/pip install in the same chunk.
3. GitHub code search fallback -- picks distinctive code lines from sampled chunks, runs =gh api search/code=, majority-votes across results. Also tries co-occurring definition names (class/function >= 8 chars). Has rate-limit retry (5 tries, exponential backoff).

Before searching, contexts are split into chunks at likely file boundaries (blank line + preamble heuristics: shebangs, docstrings, cmake, markdown headers). Each chunk extracts definition names.

** DONE Context size, compaction, and logging
CLOSED: [2026-03-01 Sun]

*** Problem
RLM's built-in prompts tell the sub-LLM that it can handle ~500K chars per =llm_query= call (hardcoded in =RLM_SYSTEM_PROMPT=, lines 12 and 40). This causes the model to generate huge string variables and verbose =print()= output, all of which accumulates in =message_history=. Each iteration can add 2-7K tokens (response + code output truncated at 20K chars). With a 32K context window, just 2-3 iterations blow past the limit before compaction kicks in.

*** Why 64K works
Compaction triggers at 85% of the context limit (~54K tokens for 64K). With 64K, the workload that hit 35K at iteration 2 is still well within limits, and when compaction does trigger the summarization call actually fits in the window. With 32K, the compaction call itself couldn't fit the full history. The 500K prompt guidance is still wrong for 64K, but the 20K char output truncation caps each iteration's damage.

- [X] =compaction=True= enabled in =RLM()= constructor.
- [X] =MODEL_CONTEXT_LIMITS[model] = max_tokens= workaround when =base_url= is set -- RLM uses substring matching on model name to find context limits, and custom endpoints (e.g. ="openai/local"=) aren't in its built-in table, so it falls back to 128K and never compacts.
- [X] =TokenLimitExceededError= caught in the main loop -- skips the example and continues.
