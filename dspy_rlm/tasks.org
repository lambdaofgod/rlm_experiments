#+title: DSPy SFT Data Collection - Tasks

* Overview

Collect SFT training data from a DSPy program (teacher LM), filter by
metric, and write JSONL in DSPy's ChatAdapter format -- ready for
finetuning with Unsloth/TRL. Based on the ~SFTDataCollector~ pattern
from [[file:dspy_sft_data_collection.org][dspy_sft_data_collection.org]]. The DSPy module type
(ChainOfThought, RLM, etc.) is configurable via YAML.

** TODO Evaluate RLM predictions via Phoenix traces

   Instead of parsing SFT JSONL output (which contains interleaved RLM
   action steps with unresolvable variable references in SUBMIT calls),
   fetch completed traces from Phoenix and extract final answers directly.

   ~evaluate.py phoenix --config_path=config.yaml --n_traces=10~

   Uses ~traces_endpoint~, ~traces_project~, and ~dataset.path~ from the
   config YAML.

*** Phoenix span schema (verified)

    Tested against local Phoenix at ~http://localhost:6006~.

    - ~from phoenix.client import Client~ / ~Client(base_url=...)~
      (~arize-phoenix-client~ package, already added)
    - ~client.spans.get_spans_dataframe(project_name=..., root_spans_only=True, limit=N)~
    - Root spans have ~name='RLM.forward'~, ~span_kind='CHAIN'~
    - Columns: ~attributes.input.value~, ~attributes.output.value~, ~status_code~, ~start_time~, etc.
    - ~status_code~: ~'OK'~ for completed, ~'ERROR'~ for failed (filter these out)
    - Input is JSON: ~{"input_args": {"context": "...", "query": "..."}}~
      - Does NOT contain dataset row ~id~ -- must match by query text
    - Output is JSON list: ~["Prediction(\n    answer='...VALUE...',\n    trajectory=[...])", [...]]~
      - ~parsed[0]~ is the Prediction repr string
      - Answer extraction: split on ~"',\n    trajectory="~, strip ~"Prediction(\n    answer='"~ prefix
      - Answers may have ~[Answer]\n~ prefix from LongBench-Pro format leak -- strip it
    - Matching to dataset: span's ~input_args.query~ corresponds to
      ~question_nonthinking~ (via prompt template).  Try direct match first;
      if no match, apply ~transform_question~ and retry (handles pre-transform
      spans).  *HACK* -- this is fragile and depends on transform_question
      being idempotent-ish.  Needs a systematic fix: either pass the row ~id~
      through to spans (e.g. as span attribute or in input), or store a
      lookup key in the dataset.
    - Duplicates: multiple spans can match the same dataset row (retries);
      group by matched ~question_nonthinking~, take the latest span per group

*** Approach

    1. Connect to Phoenix: ~Client(base_url=traces_endpoint)~
    2. Fetch root spans: ~client.spans.get_spans_dataframe(project_name=..., root_spans_only=True, limit=N)~
    3. Filter to ~status_code == 'OK'~ only
    4. For each span, extract answer from ~attributes.output.value~ (parse JSON list, split Prediction repr)
    5. Match to dataset row by comparing query text to ~question_nonthinking~ column
    6. Deduplicate: keep most recent span per dataset row (by ~start_time~)
    7. Evaluate using existing LongBench-Pro metric functions

*** Implementation: refactor ~_score_predictions~ into helpers

    Current ~_score_predictions~ does scoring, aggregation, and printing
    all in one function.  Break it into:

    1. ~_score_row(row)~ -- scores a single merged DataFrame row.
       Returns a dict (~id~, ~gold~, ~pred~, ~metric~, ~task~, ~score~)
       or ~None~ if skipped (no metric or NaN score).
    2. ~_aggregate(results)~ -- takes the list of per-row result dicts,
       groups scores by task/metric/length/difficulty, returns a summary
       dict with ~per_task~, ~per_metric~, ~per_length~, ~per_difficulty~,
       ~overall~ keys.
    3. ~_print_summary(summary, results, show_errors, n_total)~ -- prints
       the summary to stdout (same output as today).
    4. ~_write_metrics(output_path, summary, results)~ -- writes the JSON
       file with ~summary~ and ~examples~ keys.
    5. ~_score_predictions~ becomes the orchestrator: merge, loop calling
       ~_score_row~, attach length/difficulty from row, then call
       ~_aggregate~, ~_print_summary~, and optionally ~_write_metrics~.
